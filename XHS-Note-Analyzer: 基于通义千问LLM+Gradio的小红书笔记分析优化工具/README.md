XHS-Note-Analyzer: 基于通义千问LLM+Gradio的小红书笔记分析优化工具

# 项目介绍

XHS-Note-Analyzer 是一款专为小红书博主、内容运营打造的**笔记分析与优化工具**，基于 Python 开发，结合大模型与可视化技术，实现小红书笔记从「核心信息提取→深度质量分析→曝光效果预测→全方位优化→结构化报告生成」的全流程自动化处理，助力提升笔记曝光量、点赞率和完读率。

**核心定位**：非商用、仅用于个人学习，聚焦小红书笔记内容优化，提供可直接落地的标题、标签、内容、排版改进建议。

# 核心功能

工具遵循「提取→分析→预测→优化→报告」五步流程，全程自动化处理，无需手动干预：

1. **核心信息提取**：自动从笔记文本中提取标题、核心内容、标签（Hashtag）、发布时间、笔记类型、核心卖点，以 Markdown 表格标准化输出。
2. **深度质量分析**：从标题吸睛度、内容结构、标签适配性、受众适配度4个核心维度，对笔记进行专业解读，给出量化评估（低/中/高）和详细依据。
3. **曝光效果预测**：基于笔记质量分析结果，结合小红书爆款规律，提供乐观、中性、悲观三种情景下的曝光量区间、点赞/收藏率、长尾曝光潜力预测。
4. **全方位优化建议**：针对笔记薄弱点，提供3个吸睛备选标题、5-8个高流量适配标签、内容精简方案、排版优化方法，所有建议可直接落地修改。
5. **结构化报告生成**：整合所有分析结果，自动生成专业的 Markdown 分析报告，包含笔记基础信息、深度解读、曝光预测、优化方案和核心结论，可直接保存使用。

# 技术栈

项目核心技术栈清晰，兼顾实用性和可扩展性，具体如下：

- **核心语言**：Python 3.11.9
- **大模型支持**：阿里云通义千问（Qwen3-Max），通过 OpenAI SDK 兼容调用
- **可视化界面**：Gradio（快速搭建Web交互界面，支持一键生成报告）
- **并发处理**：Python Threading（多线程框架，提升处理效率）
- **输出格式化**：Markdown（标准化报告输出，提升可读性）
- **环境配置**：os 模块（读取环境变量，安全管理 API Key）

# 前置准备

## 阿里云通义千问 API Key 配置

工具依赖阿里云通义千问 API 实现大模型调用，需提前准备 API Key 并配置：

1. 访问阿里云百炼：https://bailian.console.aliyun.com/，注册并登录账号；
2. 在控制台中创建 API Key（需实名认证，有免费额度，足够个人学习使用）；
3. 配置 API Key：       
   1. 方式1（推荐）：设置系统环境变量，变量名为 `DASHSCOPE_API_KEY`，值为你的 API Key；
   2. 方式2：直接在代码中替换 `os.getenv("DASHSCOPE_API_KEY")` 为你的 API Key（不推荐，存在密钥泄露风险）。
